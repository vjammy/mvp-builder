# /mvp-builder-phase

## Purpose
Load the current phase packet and prepare for implementation.

## When to use
- When starting work on a new phase
- When resuming work after a break
- When handing off to another agent

## Files to read
- `00_PROJECT_CONTEXT.md`
- `01_CONTEXT_RULES.md`
- `AGENTS.md`
- `OPENCODE_START_HERE.md`
- Current phase `PHASE_BRIEF.md`
- Current phase `ENTRY_GATE.md`
- Current phase `TEST_PLAN.md`
- Current phase `OPENCODE_BUILD_PROMPT.md`
- Previous phase `HANDOFF_SUMMARY.md`, when applicable
- `repo/manifest.json`
- `repo/mvp-builder-state.json`

## What to do
1. Read the current phase packet files.
2. Confirm the entry gate passes.
3. Identify the implementation files to change.
4. Restate the phase goal in your own words.
5. Begin implementation only after confirming the gate.

## What not to do
- Do not skip the entry gate.
- Do not start implementation before reading the phase packet.
- Do not modify future phase files.

## Success criteria
- Phase goal is restated clearly.
- Entry gate is confirmed or blockers are surfaced.
- Implementation file list is identified.
